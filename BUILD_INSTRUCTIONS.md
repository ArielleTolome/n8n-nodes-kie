# Инструкция по сборке и запуску n8n с кастомными узлами AI Hub

Эта инструкция описывает правильный процесс сборки Docker-образа и запуска контейнера n8n, чтобы сохранить пользовательские данные (workflow, credentials) и при этом корректно обновить кастомные узлы.

## 1. Подготовка и сборка пакета

Перед сборкой Docker-образа необходимо скомпилировать TypeScript код узлов и создать npm-пакет.

1.  Увеличьте версию в `package.json` (если были изменения в коде).
2.  Выполните сборку и упаковку:

```bash
npm run build && npm pack
```

Это создаст файл вида `myspacet_ai-n8n-nodes-ai-hub-1.0.XX.tgz` в корне проекта.

## 2. Обновление Dockerfile

Убедитесь, что `Dockerfile` ссылается на актуальную версию созданного `.tgz` файла:

```dockerfile
# Пример
COPY myspacet_ai-n8n-nodes-ai-hub-1.0.20.tgz /home/node/.n8n/custom/
RUN npm install myspacet_ai-n8n-nodes-ai-hub-1.0.20.tgz
```

## 3. Сборка Docker-образа

Соберите образ с флагом `--no-cache`, чтобы Docker гарантированно подхватил новый `.tgz` файл:

```bash
docker build --no-cache -t n8n-custom-ai-hub .
```

## 4. Правильный запуск контейнера (ВАЖНО!)

Чтобы не потерять данные (схемы, доступы) и при этом увидеть новые узлы, **НЕЛЬЗЯ** монтировать всю папку `~/.n8n` целиком, так как это перекроет папку `custom` внутри контейнера, где лежат наши узлы.

Монтируйте только файлы конфигурации и базы данных:

```bash
# Остановить и удалить старый контейнер
docker stop n8n-custom && docker rm n8n-custom

# Запустить новый с правильными томами
docker run -d \
  --name n8n-custom \
  -p 5678:5678 \
  -v /Users/tinarum/.n8n/config:/home/node/.n8n/config \
  -v /Users/tinarum/.n8n/database.sqlite:/home/node/.n8n/database.sqlite \
  -v /Users/tinarum/.n8n/binaryData:/home/node/.n8n/binaryData \
  n8n-custom-ai-hub
```

**Примечание:** Путь `/Users/tinarum/.n8n` замените на ваш реальный путь к данным n8n, если он отличается.

## Решение проблем

Если вы не видите обновлений узлов в интерфейсе n8n:
1.  Убедитесь, что вы не примонтировали папку `custom` или `nodes` из локальной системы поверх контейнера.
2.  Попробуйте полностью перезагрузить страницу n8n в браузере (Ctrl+R / Cmd+R), чтобы сбросить кэш фронтенда.
